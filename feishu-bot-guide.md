# 飞书 Bot 零基础接入教程

> 从零开始，在飞书开放平台创建应用、启用机器人、配置权限、接入 OpenClaw。
> 不需要公网 IP，不需要配置回调 URL，全程 WebSocket 长连接。

---

## 前置准备

- 一个飞书账号（个人或企业均可，免费注册）
- 已经在服务器上安装好 OpenClaw Gateway
- 大约 15 分钟

---

## 第一步：登录飞书开放平台

1. 浏览器打开 [飞书开放平台](https://open.feishu.cn/)
2. 点击右上角 **「登录」**，用飞书扫码或账号密码登录
3. 首次登录可能需要填写开发者信息（姓名、邮箱等），如实填写即可

![飞书开放平台首页](https://open.feishu.cn/)

---

## 第二步：创建企业自建应用

1. 登录后进入 **[开发者后台](https://open.feishu.cn/app)**
2. 点击 **「创建应用」** 按钮
3. 选择 **「企业自建应用」**（不要选"应用商店应用"或"第三方应用"）
4. 填写应用信息：
   - **应用名称**：随便填，例如 `OpenClaw 助手` 或 `我的 Bot`
   - **应用描述**：随便填，例如 `个人 AI 助手`
   - **应用图标**：可上传任意图片或不传
5. 点击 **「创建」**

> **关键概念**："企业自建应用"就是你自己用、不对外发布的应用，不需要飞书审核，适合个人使用。

---

## 第三步：获取 App ID 和 App Secret

创建成功后自动进入应用详情页：

1. 左侧菜单点击 **「凭证与基础信息」**
2. 你会看到：
   - **App ID**：形如 `cli_a1b2c3d4e5f6g7h8`
   - **App Secret**：形如 `pQrStUvWxYz1234567890AbCdEfGhIjKlMn`

3. 点击 App Secret 旁边的 **「查看」** 按钮，可能需要飞书验证（扫码或密码）
4. **把这两个值记下来**，接下来部署时会用到

> **安全提示**：App Secret 相当于密码，不要发给别人，不要传到公开仓库。OpenClaw 会把它存在权限 600 的文件里。

---

## 第四步：添加机器人能力

1. 左侧菜单点击 **「应用功能」** → **「机器人」**
2. 点击 **「启用」** 开关
3. 机器人配置保持默认即可，无需修改
4. 点击 **「保存」**

> 启用后，飞书用户才能搜索到这个机器人并发消息。

---

## 第五步：配置事件订阅（WebSocket 模式）

这一步让飞书把用户消息实时推送给 OpenClaw。

1. 左侧菜单点击 **「事件订阅」**
2. **「Encrypt Key」** 保持默认即可（自动生成）
3. **「请求方式」** 选择 **「使用长连接（WebSocket）接收事件」**

   > ⚠️ **不要选"使用 HTTP 接收事件"**。HTTP 模式需要一个公网可访问的回调 URL，
   > WebSocket 模式不需要公网 IP，OpenClaw 主动连接到飞书服务器。

4. 不需要填写 Request URL

### 5.1 添加事件

在「事件订阅」页面下方，点击 **「添加事件」**：

1. 在弹出窗口中搜索 `im.message.receive_v1`
2. 勾选它，点击 **「确认添加」**

   > 这个事件表示"收到消息"，是机器人接收用户消息所必需的。

3. 添加完成后，页面应该显示已订阅 1 个事件

---

## 第六步：配置权限（Scopes）

机器人需要权限才能收发消息。

1. 左侧菜单点击 **「权限管理」**
2. 在 **「API 权限」** 标签页，点击 **「添加权限」**

### 必须添加的权限（最小集）：

| 权限名称 | 说明 | 搜索关键词 |
|----------|------|-----------|
| `im:message` | 获取消息 | `im:message` |
| `im:message:send_as_bot` | 以机器人身份发消息 | `send_as_bot` |
| `im:chat` | 获取群聊信息（可选，群聊功能需要） | `im:chat` |
| `im:chat:send_as_bot` | 在群聊中以机器人身份发消息（可选） | `chat:send_as_bot` |

**操作步骤：**
1. 在权限搜索框输入关键词
2. 找到对应权限，点击 **「添加」**
3. 重复直到以上权限全部添加

### 可选权限（如需图片/文件处理）：

| 权限名称 | 说明 | 搜索关键词 |
|----------|------|-----------|
| `im:image` | 读取消息中的图片 | `im:image` |
| `im:file` | 读取消息中的文件 | `im:file` |
| `im:resource` | 下载消息中的多媒体 | `im:resource` |

### 权限审批

- 个人飞书租户：添加的权限会自动生效，无需审批
- 企业飞书租户：部分权限需要管理员审批。添加后会显示 **「待审批」**，联系企业管理员通过即可

---

## 第七步：发布应用（或添加测试用户）

### 方式一：发布应用（推荐）

1. 左侧菜单点击 **「应用发布」** → **「版本管理与发布」**
2. 点击 **「创建版本」**
3. 填写版本号（如 `1.0.0`）和更新说明（如 `首次发布`）
4. 点击 **「保存」** → **「申请发布」**
5. 个人飞书租户会自动通过；企业租户可能需要管理员审批

### 方式二：仅添加测试用户（不发布）

如果不想走发布流程（或等待审批期间测试）：

1. 左侧菜单点击 **「安全设置」**
2. 找到 **「可用范围」** 区域
3. 点击 **「添加测试用户/部门」**
4. 选择你自己的飞书账号
5. 保存

> 测试用户模式下，只有被添加的用户才能搜索到和使用机器人。

---

## 第八步：在飞书中使用机器人

### 8.1 找到机器人

1. 打开飞书客户端（桌面或手机）
2. 在搜索框搜索你的应用名称（第二步填的名称）
3. 在搜索结果中点击机器人，进入对话

### 8.2 先不要发消息

此时飞书侧已配置完毕，但消息还不会被处理，因为 OpenClaw 还没有配置飞书通道。

---

## 第九步：在 OpenClaw 中接入飞书

回到部署流程，在 **G. 飞书 Bot 参数** 中填写：

```bash
CONFIGURE_FEISHU="true"

FEISHU_DOMAIN="feishu"           # 中国大陆用 feishu
FEISHU_CONNECTION_MODE="websocket"
FEISHU_APP_ID="cli_xxxxxx"      # 第三步记录的 App ID
FEISHU_APP_SECRET="xxx..."      # 第三步记录的 App Secret
FEISHU_DM_POLICY="pairing"
FEISHU_GROUP_POLICY="allowlist"
FEISHU_REQUIRE_MENTION="true"
```

填写后执行飞书部署阶段（阶段 10），OpenClaw 会自动连接飞书。

---

## 第十步：配对（Pairing）并测试

飞书通道默认使用 pairing 策略，首次使用需要配对：

### 10.1 触发配对

1. 在飞书客户端给机器人发任意消息（例如 `hello`）
2. 机器人会返回一个 **配对码**（6 位字母数字）

### 10.2 批准配对

在服务器上执行：

```bash
export PATH="/usr/local/bin:$HOME/.local/npm-global/bin:$PATH"

# 查看待审批的配对请求
openclaw pairing list feishu

# 批准配对（替换 CODE 为机器人返回的配对码）
openclaw pairing approve feishu <CODE>
```

### 10.3 验证

1. 再次给机器人发消息 `用一句话回复：飞书已接入`
2. 机器人应该正常回复

### 10.4 测试群聊（可选）

1. 在飞书创建一个测试群
2. 把机器人拉进群
3. **不 @机器人**，发一条消息 → 机器人应该不回复
4. **@机器人** 发 `/status` → 机器人应该回复状态

## 常见问题排查

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 搜索不到机器人 | 应用未发布/未加测试用户 | 检查第七步 |
| 机器人不回复 | 事件订阅未启用 | 检查第五步是否选了 WebSocket + 添加了事件 |
| 机器人不回复 | 权限不足 | 检查第六步权限是否全部添加 |
| 机器人不回复 | WebSocket 断开 | 服务器执行 `openclaw gateway restart` |
| 配对码不出现 | DM 策略配置错误 | 检查 `FEISHU_DM_POLICY=pairing` |
| 群聊中 @也不回复 | groupAllowFrom 拦截 | 获取群 chat_id 加入 allowlist |
| App Secret 错误 | 复制不完整 | 重新复制，确认没有多余空格 |

### 获取群 chat_id

群聊不响应时，可能需要把群加入 allowlist：

1. 在群聊中 @机器人 发任意消息
2. 在服务器查看日志：
   ```bash
   openclaw logs --follow
   ```
3. 日志中搜索 `chat_id` 或 `oc_` 开头的字符串
4. 将群 ID 加入配置：
   ```bash
   openclaw config set channels.feishu.groupAllowFrom '["oc_xxxxxxxx"]' --strict-json
   openclaw gateway restart
   ```

## 权限速查表

飞书开放平台左侧菜单 → 需要操作的页面汇总：

```
📋 飞书开放平台
├── 凭证与基础信息    → 获取 App ID / App Secret
├── 应用功能
│   └── 机器人       → 启用机器人
├── 事件订阅          → 选 WebSocket，订阅 im.message.receive_v1
├── 权限管理          → 添加 im:message + send_as_bot 等权限
├── 安全设置          → 添加测试用户（发布前测试用）
└── 应用发布
    └── 版本管理与发布 → 创建版本并发布
```

## 与微信对比

| | 微信 (Weixin) | 飞书 (Feishu) |
|------|-------------|------------|
| 接入方式 | 扫码登录 | App ID + App Secret |
| 登录态 | 持久化（插件本地存储） | 永久（凭据认证） |
| 群聊 | 支持，allowlist | 支持，allowlist |
| 公网要求 | 不需要 | 不需要（WebSocket） |
| 初次配置难度 | 低（扫码即可） | 中（需创建应用、配权限） |
| 稳定性 | 依赖扫码登录态 | 高（API 凭据） |
| 适用场景 | 个人微信日常使用 | 企业/团队协作 |
